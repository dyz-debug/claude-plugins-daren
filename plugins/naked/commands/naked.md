# /naked - 가벼운 모드로 전환

자료조사 · 일반 질의 · 짧은 답변용 모드. 프로젝트 맥락 무시, 짧고 직접적인 응답, 도구 호출 최소화.

## 한계 (이 모드의 실제 효과 / 한계)

슬래시 커맨드는 이미 시작된 세션 안에서 작동하므로 **시스템 프롬프트와 CLAUDE.md는 컨텍스트에 그대로 남아있음**. 입력 토큰 자체는 안 줄어듭니다.

| 항목 | 효과 |
|---|---|
| 출력 토큰 절감 (짧은 답변) | ✅ |
| 도구 호출 최소화 → 컨텍스트 폭증 방지 | ✅ |
| 프로젝트 맥락(BEFORE/WHEN 트리거·특정 가이드라인) 의도적 무시 | ✅ |
| 입력 토큰 절감 (시스템 프롬프트·CLAUDE.md 제거) | ❌ |

→ **진짜 입력 토큰 절감 원하면** 새 세션을 가벼운 환경(프로젝트 외부)에서 시작하세요.

## 행동 원칙 (이 모드 동안 지킬 것)

1. **짧고 직접적인 답변** — 불필요한 헤더·표·구조화 자제. 핵심만 1~3문장. 표나 목록은 비교가 진짜 필요할 때만.
2. **도구 호출 최소화** — `WebFetch` / `WebSearch` / `Read`만 사용 (그것도 꼭 필요할 때). `Bash` · `Edit` · `Write`는 사용자가 명시 요청할 때만.
3. **프로젝트 맥락 의도적 무시** — `CLAUDE.md`의 BEFORE/WHEN 트리거(특정 톤·가이드·클라이언트 정보 등) 참조하지 않음. 프로젝트 하위 설정 파일 안 읽음.
4. **추측은 명확히 표시** — 확인 안 된 정보는 "추측" 또는 "확인 안 됨"으로 라벨. 환각 방지.
5. **자료조사 우선** — 사용자 질문이 정보 검색 성격이면 `WebSearch`로 최신 정보 가져오기 (지식 컷오프 이전 정보만 답하지 말 것).
6. **마크다운 링크 사용 자제** — 파일 경로 링크 등은 풀모드에서 유용하지만 가벼운 답변엔 시각적 잡음.

## 자동 해제 조건

다음 상황이면 가벼운 모드 자동 해제하고 평소 작동(CLAUDE.md 트리거 기반)으로 복귀:

- 사용자가 본인 프로젝트 작업 요청 (실제 코드 수정·메시지 작성·리포트 등)
- 사용자가 명시적으로 "보통 모드" · "풀모드" · "끝" · "naked 해제" 요청
- 프로젝트별 전용 도구 사용이 필요한 작업

해제 시점에 사용자에게 한 줄 알림: `[naked 모드 해제됨 — 평소 작동으로 돌아갑니다]`

## 발동 직후 첫 응답 형식

이 슬래시 입력 직후 첫 응답은 다음 한 줄로만:

```
[naked 모드 진입 — 짧고 직접적, 도구 최소, 프로젝트 맥락 무시]
```

그 뒤로 사용자 질문 받으면 위 행동 원칙 따라 답변.
